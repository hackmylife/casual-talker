# UI/UX設計計画: AI English Conversation Trainer (MVP)

## 0. 設計の大前提

このアプリの本質は「初心者が安心して話せるように設計された会話体験」である。UI/UXのあらゆる判断は以下の原則に従う。

- **迷わせない**: 画面あたりの選択肢を最小限にする。次にやることが常に1つだけ明確に見える
- **怖がらせない**: 試験的な雰囲気を排除し、温かみのあるビジュアルで安心感を与える
- **待たせない**: AIの応答待ち時間を「不安」ではなく「期待」に変えるフィードバック設計
- **UI言語は原則日本語**: ナビゲーション、ボタンラベル、説明文は全て日本語。英語はAIの発話と学習コンテンツのみ

## 1. 各画面のレイアウト設計方針（モバイルファースト）

### 全画面共通の方針
- ビューポート: `375px`(iPhone SE)を基準設計。最大`428px`(iPhone 14 Pro Max)まで自然にスケール
- セーフエリア: `env(safe-area-inset-*)` を全画面で考慮（ノッチ、ホームインジケータ）
- 1カラムレイアウト。横スクロールなし
- タッチターゲット: 最小`48x48px`（WCAG準拠）、主要ボタンは`56px`以上
- 画面下部にナビゲーションバー（ホーム / 履歴 の2タブのみ）

### 英語が表示される箇所（明示リスト）
- セッション画面: AIの発話バブル内テキスト
- セッション画面: ユーザーの発話バブル内テキスト（STT結果）
- 結果画面: 自然な表現の例文
- 結果画面: 復習フレーズ
- ホーム画面: テーマカード内の英語テーマ名
- 履歴画面: セッションカード内の英語テーマ名

上記以外は全て日本語で表示する。

### 主要コンポーネントの仕様

**Button** - 3つのvariant:
- `primary`: 塗りつぶし。CTA用。高さ56px、角丸16px
- `secondary`: アウトライン。サブアクション用
- `ghost`: テキストのみ。リンク的操作用

**ChatBubble** - 2つのvariant:
- `ai`: 左寄せ、背景色あり、音声再生ボタン付き
- `user`: 右寄せ、アクセントカラー背景

**VoiceInputButton** - 3つの状態:
- `idle`: マイクアイコン + 「話す」テキスト
- `recording`: 波形アニメーション + 赤い録音インジケータ
- `processing`: ローディングスピナー

## 3. インタラクションデザイン

### 3.1 音声入力UXフロー

```
[タップ] → [録音開始] → [話す] → [タップ or 無音検出] → [処理中] → [テキスト表示] → [AI応答]
```

**詳細設計:**

1. **録音開始**: ボタンタップで即座に開始。トグル方式（押しっぱなしではなく）。理由: 長文を話す際に指が疲れない
2. **録音中の視覚フィードバック**: ボタンが赤くパルスし、音声波形がリアルタイム表示
3. **録音停止**: もう一度タップ、または3秒の無音で自動停止。自動停止前に「まだ話していますか？」とソフトにプロンプト
4. **処理中**: タイピングインジケータ（`...`のアニメーション）+ 「聞き取り中」ラベル
5. **テキスト表示**: 認識テキストをユーザーバブルに表示。「言い直す」ボタンを一時的に表示（3秒後にフェードアウト）

**マイク権限リクエスト:**
- オンボーディング Step 2 で「マイクを使って会話します」と案内してからブラウザ権限ダイアログを表示
- 拒否された場合はテキスト入力フォールバックUIを表示

### 3.2 詰まり救済UIの動線

**受動的救済（自動検出）:**
```
録音開始 → 5秒間発話なし → 「ヒントを見る？」がスライドイン
         → 10秒間発話なし → 「大丈夫！こんな風に言えるよ」と回答例カードが表示
         → 15秒間発話なし → 「スキップして次に進む？」を提案
```

**能動的救済（ボタン操作）:**
- ボタンラベルは日本語（「ヒント」「回答例」「もう一度聞く」）
- アイコン付きで直感的に機能が分かる

**回答例の段階表示（3ステップ）:**
1. 日本語でのヒント: 「週末にしたことを答えてみよう」
2. 英語キーワード: 「went / park / fun」
3. 完全な回答例: 「I went to the park. It was fun.」

## 5. アニメーション/トランジション

### 基本原則
- アニメーションは「意味のあるもの」のみ
- `prefers-reduced-motion`で全アニメーション無効化可能
- 持続時間は150ms~300ms

### 具体的なアニメーション

| 対象 | アニメーション | 持続時間 |
|------|-------------|---------|
| ChatBubble出現 | 下からスライドイン + フェード | 250ms |
| VoiceInputButton録音中 | パルス（スケール1.0→1.05） | 1000ms loop |
| TypingIndicator | 3ドットの順次バウンス | 600ms loop |
| ヒントボトムシート | 下からスライドアップ | 300ms |
| CTAボタンタップ | スケール縮小→戻り | 150ms |
| ページ遷移 | フェード + 軽いスライド | 300ms |

## 6. アクセシビリティ

### WCAG 2.1 AA準拠目標

| 項目 | 対応方針 |
|------|---------|
| コントラスト比 | テキスト4.5:1以上 |
| タッチターゲット | 最小48x48px |
| フォーカス管理 | `:focus-visible`使用 |
| スクリーンリーダー | 全要素に`aria-label`、録音状態は`aria-live="polite"` |
| 色に依存しない | 録音中は色+アイコン+テキストの3重フィードバック |
| 言語属性 | 英語テキストに`lang="en"`付与 |

## 7. UIライブラリ

| カテゴリ | 推奨 | 理由 |
