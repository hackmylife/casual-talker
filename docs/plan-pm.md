## 1. MVPの機能優先度と段階的リリース戦略

### 優先度マトリクス

| 優先度 | 機能 | 理由 |
|--------|------|------|
| **P0 (Must)** | セッション画面（AI会話 + 音声入出力） | コア体験そのもの |
| **P0 (Must)** | テーマ選択（固定8テーマ） | 会話の起点 |
| **P0 (Must)** | 詰まり救済（ヒント・回答例） | PRDの設計原則の最優先事項 |
| **P0 (Must)** | 結果画面（フィードバック表示） | セッション完了の動機づけ |
| **P1 (Should)** | ホーム画面（おすすめテーマ・復習ポイント） | 継続利用の導線 |
| **P1 (Should)** | 学習履歴保存・履歴画面 | 継続率向上 |
| **P1 (Should)** | アナリティクス（イベントトラッキング + 1問アンケート） | MVP価値検証に必須 |
| **P2 (Nice)** | 難易度自動調整（Lv1-5） | 初回はLv1固定でも成立 |
| **P2 (Nice)** | 復習フレーズの次回セッション活用 | パーソナライズの種 |

### 段階的リリース戦略

**Phase 1: 音声会話コアループ（Day 1-8）**
- テーマ選択 → AI会話（音声入出力） → フィードバック表示
- 認証あり（Supabase Auth）、DB永続化あり
- 目的: コア音声会話体験の検証（テキストのみのWalking Skeletonでは価値検証不可）

**Phase 2: 体験の充実（Day 9-14）**
- 詰まり救済機能（ヒント、リピート、日本語補助、回答例）
- ホーム画面、履歴画面
- 初回オンボーディングフロー
- アナリティクス基盤

**Phase 3: MVP完成（Day 15-18）**
- 難易度制御ロジック
- UIポリッシュ（スマホ最適化の仕上げ）
- パフォーマンス最適化（応答速度）
- iOS Safari手動テスト

## 2. ユーザーストーリーの整理

### Phase 1 のユーザーストーリー（最重要）

```
US-1: ユーザーとして、テーマを選んでAIと英会話セッションを開始したい
  受入条件:
  - 8テーマから1つ選択できる
  - セッション開始後、AIが最初に英語で話しかける
  - 5-8ターンの会話ができる
  - セッション終了後、結果画面に遷移する

US-2: ユーザーとして、音声で英語を話して会話したい
  受入条件:
  - マイクボタンをタップして音声入力できる
  - 音声がテキストに変換されて表示される
  - AIの応答が音声で再生される

US-3: ユーザーとして、セッション後にフィードバックを確認したい
  受入条件:
  - できたこと、改善ポイント、自然な表現、復習フレーズが表示される
  - やさしいトーンで否定的表現がない
  - 点数評価は表示されない
```

### Phase 2 のユーザーストーリー

```
US-4: ユーザーとして、詰まったときにヒントや回答例を見たい
US-5: ユーザーとして、過去のセッション履歴を確認したい
US-6: ユーザーとして、ホーム画面でおすすめテーマを確認したい
US-7: ユーザーとして、初回利用時に使い方を理解したい（オンボーディング）
```

### スコープ外（MVPに含まない）
- 発音評価
- ゲーミフィケーション（バッジ、ストリーク等）
- 複数コース
- 課金・サブスクリプション
- ソーシャル機能

## 4. データモデル設計の方針

### 基本方針
- セッションログ中心設計（PRDの技術方針に従う）
- LLMへのプロンプト構築に必要な情報を効率的に取得できる構造にする

### スキーマ概要

```
User: id, supabase_uid, displayName, level(default:1), createdAt
Course: id, title, description, sortOrder（MVPでは1レコード）
Theme: id, courseId, title, description, targetPhrases, baseVocabulary, sortOrder（MVPでは8レコード）
Session: id, userId, themeId, difficulty, status, startedAt, endedAt, turnCount
Turn: id, sessionId, turnNumber, aiText, userText, hintUsed, repeatUsed, jaHelpUsed, exampleUsed
Feedback: id, sessionId, achievements, naturalExpressions, improvements, reviewPhrases
PhraseProgress: id, userId, phrase, translationJa, timesUsed, timesStruggled, masteryLevel
```

## 5. MVP成功指標の測定方法

### 定量指標

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| セッション完了率 | 70%以上 | Session.status = completed / 全セッション |
| 3日継続率 | 40%以上 | 初回から3日以内に再セッション |
| 7日継続率 | 25%以上 | 初回から7日以内に3回以上 |
| 平均発話ターン数 | 5ターン以上 | Turnテーブルのカウント |
| ヒント使用率 | 30-60% | 高すぎると難、低すぎると易 |
| セッション平均時間 | 5-12分 | Session.durationSeconds |

### 定性指標
- セッション後の1問アンケート（「今日の会話は楽しかったですか？」4段階）
- ユーザーインタビュー（5-10名）

### 計測基盤
- Phase 2でイベントトラッキング（セッション開始/完了/離脱、救済機能使用率）を実装
- DBのSession/Turnデータから集計SQLで算出

### 危険信号
- セッション完了率50%以下 → 難易度 or UI問題
- 平均ターン数3以下 → 早期離脱
- ヒント使用率80%以上 → Lv1が難しすぎる
- 音声入力エラー率20%以上 → STT品質問題

## 6. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 音声パイプライン合計レイテンシ > 3秒 | 致命的 | TTS先行リクエスト、レイテンシバジェット管理、Gate 2で実測検証 |
| iOS Safari の音声入力不安定 | 高 | テキスト入力フォールバック常備、iOS実機手動テスト |
| LLM応答品質のブレ | 中 | システムプロンプトの入念な設計・テスト |
| 初回ユーザーの離脱 | 中 | オンボーディングフロー（Phase 2） |
